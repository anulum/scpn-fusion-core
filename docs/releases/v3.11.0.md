# SCPN Fusion Core v3.11.0

Release date: 2026-07-17

## Scope

v3.11.0 is a minor feature release on top of v3.10.1. It adds new control,
transport, surrogate, and equilibrium capabilities, lands a large
modular-decomposition pass, and extends the validation-evidence surfaces. No
public API is removed and all additions are backward compatible with v3.10.1.
The release does not add any full-fidelity solver-claim acceptance and does not
change the fail-closed evidence boundary.

## What changed

### Added

- Safety-shield composition with a real constrained-PPO control policy (M-4).
- Real TGLF transport surrogate fit (F-7).
- Petri→SNN interlock with proven replay invariance and an exported proof hash
  (M-1).
- Rust design evaluator brought to Python parity and dispatched (design-scan).
- Fastest-first dispatch tiers for the FNO turbulence (`PyFnoController`),
  surrogate-MPC (`PyMpcController`), and runaway-electron Fokker-Planck
  (`PyFokkerPlanckSolver`) kernels.
- Per-surrogate UQ/OOD cards with a fail-closed drift guard (T-4) and a safety
  traceability matrix generator with a fail-closed drift guard (T-2).
- PB-KBM pedestal constraint-loop tier with a tier benchmark against digitised
  EPED1 references (F-5). The benchmark reports documented divergence and makes
  no parity claim.
- Solov'ev exact equilibrium suite with a published psi_N gate and per-file
  attribution (F-1).

### Changed

- Modular decomposition of oversized modules by responsibility
  (`fusion_ignition_sim`, `blanket_neutronics`, `pretrained_surrogates`,
  `impurity_transport`, `hil_harness`, `_multi_compat`, `frc_rigid_rotor`) and
  extraction of native-trust primitives from `hpc_bridge`.
- Subsystem and control/core demos routed through `logging`; stale strict-typing
  suppressions narrowed or removed.

### Fixed

- Security: bumped `pillow` and `click` across the hash-pinned locks (pip-audit
  clean on the tracked requirement inputs and the local release virtualenv).
- Transport: implicit electron-ion exchange with an honest RHS and provenance
  manifest; dt-independent steady state with no limit cycle.
- Training: device-resident per-batch indexing to avoid GPU OOM; batched final
  validation metrics with best-on-improvement checkpointing.
- Documentation lock pins numpy 1.26.4 for the Python 3.12 hashed install.

## Evidence boundary

The full-fidelity campaign remains fail-closed. GENE/CGYRO/GS2 nonlinear
gyrokinetic parity, full electromagnetic parity, DREAM kinetic parity,
independent mechanistic Aurora/STRAHL recycling validation, and production
MPI/multi-GPU scaling still require same-case external outputs, provenance,
checksums, thresholds, grid or scaling evidence, and native comparisons before
acceptance. The PB-KBM pedestal tier (F-5) is reported as documented divergence
against EPED1 with no parity claim.
