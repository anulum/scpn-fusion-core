Enforce the HIL actuator boundary: drive the plant with the DAC-applied value, fail-safe on non-finite

The HIL control loop clamped the actuator command inside SensorInterface.write_dac
(the DAC's +/-10 V range plus slew-rate limit) but discarded the return and advanced
the plant on the RAW controller command (hil_control_loop.py). The DAC clamp was real
but bypassed: a controller output of 1e6 reached the plant unchanged. Separately,
write_dac latched a non-finite command into _last_dac_voltage, so one NaN sample
permanently poisoned every subsequent output.

Make write_dac the single enforced actuator boundary:
- A non-finite command (NaN/inf) is a fault the converter cannot emit; hold the last
  valid output (fail-safe hold), count it in a new dac_faults attribute, and never
  latch it into the converter state.
- The control loop now advances the plant on the value write_dac actually applied
  (clamped, slew-rate limited, fault-held), never the raw command.

A 1e6 command now saturates at the +/-10 V DAC range at the plant; a NaN command is
held and flagged, so the plant state stays finite. Both source modules stay at 100%
line coverage; four new tests cover the fail-safe hold (no poison after a NaN/inf),
the clamp reaching the plant, and NaN never advancing the plant.

Closes KIMI due-diligence finding K17-A1 (HIL clamp-discard) and the write_dac arc of
K17-A2 (NaN-latching across the Python safety stack).
