Reject non-finite inputs at the free-boundary safety supervisor boundary

FreeBoundarySafetySupervisor.filter_action only finite-checked the geometry vector
(predicted_next_state). A NaN or inf in any other channel -- proposed_action,
coil_currents, corrected_state, target_state, or bias_hat -- passed straight through:
np.clip preserves NaN, so a NaN proposed action came back verbatim in the emitted
command (demonstrated: proposed [nan, 0.1, 0, 0] returned action [nan, 0.1, 0, 0]).
The supervisor is the last gate before commands reach the coils, so a non-finite
sample must be rejected here, not silently clipped into a plausible-looking action.

Add a finite-guard over all five safety-relevant input arrays immediately after the
length check, raising ValueError on any non-finite value -- consistent with the
existing geometry finiteness contract. Nominal (all-finite) calls are unaffected.

Module stays at 100% line coverage; three new guard tests cover a non-finite proposed
action, coil current, and measured state.

Closes the safety-supervisor arc of KIMI due-diligence finding K17-A2 (NaN-latching /
passthrough across the Python safety stack).
