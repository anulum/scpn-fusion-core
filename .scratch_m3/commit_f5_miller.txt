Add Miller shaped-geometry ballooning second-stability via pyrokinetics (F-5)

The PB-KBM pedestal tier evaluated the ballooning boundary in reduced circular
s-alpha geometry (first stability only), which collapses to the scan floor for
strongly shaped DIII-D pedestals because it lacks the second-stability access
that real pedestals exploit. That was the recorded F-5 blocker
(`shaped_geometry_miller_ballooning_required_for_second_stability_access`).

Close it by integrating the real published infinite-n ideal ballooning solver
shipped in pyrokinetics (`Diagnostics.ideal_ballooning_solver`, adapted from
R. Gaur's ideal-ballooning-solver) over a Miller local equilibrium — the same
"run a real reference code in a dedicated venv and commit a provenance'd
artifact" pattern already used for the DREAM (F-3) and TORAX (F-4) references.

- `validation/ballooning_reference_runner.py` runs inside a dedicated
  `.venv-pyrokinetics` (pyrokinetics pulls numpy>=2, incompatible with the
  project pins). It sweeps the MHD ballooning drive `alpha = -q^2 (R0/a) beta'`
  at fixed magnetic shear to locate the first-stability boundary and to detect
  second-stability access, over the DIII-D shaping (kappa=1.74, delta=0.3, plus
  pedestal-representative shaping gradients and Shafranov shift) with a
  near-circular self-check row. The artifact carries full provenance
  (pyrokinetics version, solver source, runner SHA-256, runtime metadata) and a
  recorded honest scope boundary (this is the ideal-MHD ballooning boundary, not
  the full ELITE peeling-ballooning eigenvalue).
- `src/scpn_fusion/core/ballooning_second_stability.py` is the hermetic consumer:
  it reads the committed artifact (never pyrokinetics) and interpolates
  `alpha_crit(shear)`, filling second-stability rows to the scan ceiling so the
  coupled peeling-ballooning boundary becomes current-limited there.
- `eped_pb_kbm.py` gains an opt-in `geometry="miller_second_stability"` that
  drives `ballooning_alpha_crit` from the reference instead of the collapsing
  circular s-alpha `find_marginal_stability`. The default `"salpha"` geometry and
  every existing contract are unchanged (the s-alpha DIII-D collapse still
  documents the blocker); the result method tag records the geometry.

Physics validated in a de-risk pass: near-circular alpha_crit rises with
magnetic shear (0.50/0.55/0.98 at shat=0.5/1.0/2.0, matching the classic s-alpha
trend), and a Shafranov shift opens second-stability access (shat=0.5 stable to
the full drive) — the mechanism F-5 needs. pyrokinetics is LGPL-3.0-or-later:
used as an unmodified library via its public API in an isolated venv, with only
the derived critical-alpha numbers redistributed (attributed) — compatible with
this AGPL-3.0 project.

Scope of this commit: it builds and hermetically tests the capability. The
one-time heavy artifact generation (pyrokinetics eigensolves) and the resulting
quantitative EPED1 DIII-D parity benchmark are deferred to an off-peak run per
the owner's workstation-load directive, so no EPED parity is claimed yet — the
committed artifact and the benchmark flip land in that follow-up.

Tests are fully hermetic (no pyrokinetics, no heavy solve): the boundary logic
runs against injected synthetic growth-rate functions, the pyrokinetics paths
against a fake `pyrokinetics` module on `sys.modules`, and the PB-KBM geometry
against injected synthetic references. 57 new/covering tests; the three source
modules are at 100% line coverage; ruff format+check, project mypy and the
package-export and reference-data-manifest gate tests are green.
