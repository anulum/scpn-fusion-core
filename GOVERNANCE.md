## Decision Process

1. **Minor changes** (typos, docs, test additions): merge after CI passes.
2. **Feature additions**: open an issue first for discussion, then PR.
3. **API changes**: require explicit approval from the project lead.
4. **Security patches**: fast-tracked, coordinated via private email.

## Maintainer Team

The `@anulum/maintainers` GitHub team has write access and is responsible
for code review, CI health, and release management.

## Code of Conduct

All participants must follow the [Code of Conduct](CODE_OF_CONDUCT.md).
